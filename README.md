This prototype demonstrates the core functionality of an in-product calendar interface. It includes a basic month and week view, event creation, and internal state management. The goal of the demo is to validate the technical feasibility of building a full calendar module inside Rocket.Chat.

Features Included in the Demo
Rendering calendar events on a simple month and week layout
A basic event creation flow using an input form
Switching between month and week views
Managing events in local component state
A minimal layout structure that mirrors the planned Rocket.Chat integration
Purpose of the Demo

The purpose of this prototype is to showcase understanding of the calendar requirements, UI structure, and event flow. It also confirms that the proposed technical approach is practical and can be expanded into a complete production-ready calendar interface during the project.

What This Demo Proves
The calendar views can be implemented cleanly with React and TypeScript
Event data can be managed, displayed, and updated reliably
The UI structure aligns with Rocket.Chat’s component architecture
The solution is technically feasible within the GSoC development timeline
